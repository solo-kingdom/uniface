# 功能示例：认证系统

## 描述
本提示词是在 uniface 项目中实现新功能的模板。它展示了请求功能实现时期望的结构和内容。

## 上下文
uniface 项目旨在统一不同系统的接口。当我们添加新功能时，需要保持代码质量、架构和文档的一致性。

## 需求

### 功能需求
1. 实现基于 JWT 的认证系统
2. 支持用户注册和登录
3. 提供保护路由的中间件
4. 包含令牌刷新机制
5. 安全存储用户凭据（哈希密码）

### 技术需求
- 使用 `golang.org/x/crypto/bcrypt` 进行密码哈希
- 使用 `github.com/golang-jwt/jwt/v5` 处理 JWT 令牌
- 遵循 `internal/core` 中的现有代码结构
- 添加适当的错误处理和日志记录
- 编写单元测试，覆盖率 >80%
- 更新文档

### 非功能需求
- 认证操作的响应时间 < 100ms
- 安全存储敏感数据
- 代码清晰可维护
- 适当的关注点分离

## 实现细节

### 需要创建/修改的文件
- `internal/core/auth/` - 认证逻辑的新包
- `internal/core/middleware/auth.go` - 认证中间件
- `pkg/auth/types.go` - 公共认证类型和接口
- `test/auth_test.go` - 单元测试

### 依赖项
- 确保与现有 Go 1.24 兼容
- 更新 `go.mod` 和 `go.sum`
